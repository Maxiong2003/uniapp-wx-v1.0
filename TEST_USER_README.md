# 测试用户功能说明

## 功能介绍
本项目添加了测试用户功能，可以绕过正常的登录流程，直接使用预设的测试用户进行应用测试。

## 配置方法

### 1. 测试用户配置文件
测试用户的配置信息存储在 `common/testUser.js` 文件中，您可以根据需要修改以下配置：

```javascript
// 测试用户配置
const testUser = {
  // 是否启用测试用户绕过登录
  enable: true,
  // 测试用户信息
  user: {
    // 测试用户的token
    token: "test-token-123456",
    // 测试用户基本信息
    userInfo: {
      userId: "test-user-001",
      nickName: "测试用户",
      portrait: "/static/wx/y01.png",
      phone: "13800138000",
      userType: "SINGLE"
    }
  }
};
```

### 2. 配置项说明

- **enable**: 是否启用测试用户功能
  - `true`: 启用测试用户，跳过登录流程
  - `false`: 禁用测试用户，使用正常登录流程

- **user.token**: 测试用户的认证令牌
  - 可以设置为任意字符串，用于模拟登录状态

- **user.userInfo**: 测试用户的基本信息
  - `userId`: 用户ID
  - `nickName`: 用户名
  - `portrait`: 用户头像路径
  - `phone`: 手机号码
  - `userType`: 用户类型（SINGLE表示个人用户）

## 使用方法

1. 确保 `testUser.js` 文件中的 `enable` 配置为 `true`
2. 启动应用，将自动跳过登录页面，直接进入主页面
3. 应用中将显示预设的测试用户信息

## 注意事项

1. **仅用于测试环境**：测试用户功能仅适用于开发和测试环境，生产环境中请确保将 `enable` 设置为 `false`

2. **模拟数据限制**：测试用户使用的是模拟数据，可能无法正常使用所有需要服务器验证的功能

3. **音视频功能**：在APP环境下，测试用户会尝试模拟音视频登录，但可能无法正常使用音视频通话功能

4. **数据隔离**：测试用户的数据与真实用户数据完全隔离，不会影响真实用户

## 关闭测试用户功能

要关闭测试用户功能，只需将 `testUser.js` 文件中的 `enable` 配置修改为 `false` 即可：

```javascript
const testUser = {
  enable: false,
  // 其他配置...
};
```

关闭后，应用将恢复正常的登录流程，用户需要输入正确的手机号和密码/验证码才能登录。
